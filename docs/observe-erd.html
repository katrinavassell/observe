Relationships

users → user_accounts ← accounts (many-to-many via role)
accounts → customers, observe_events, subscriptions, plans, integrations, sdk_api_keys (owns all data)
customers ↔ observe_events (via account_id + customer_id)
customers ↔ subscriptions (via account_id + customer_id)
subscriptions → plans (via plan_id)
alert_rules → alert_history (ON DELETE CASCADE)
custom_cohorts → cohort_members (ON DELETE CASCADE)
routing_configs → routing_targets, routing_rules (ON DELETE CASCADE)
feature_pricing ↔ feature_definitions (via feature_key)

Stripe Revenue Flow

Connect Stripe API key
  → syncStripeDataForUser()
    → Fetch customers, subscriptions, plans from Stripe API
    → Detect pricing_model per subscription (flat/metered/tiered/hybrid)
    → Store in DB: customers + subscriptions + plans
    → runRevenueBackfill() — auto-enriches existing events

SDK Event Ingest → POST /events/ingest
  → Auto-create customer record (with stripe_customer_id from meta)
  → Revenue priority:
     1. Explicit revenue_amount (highest)
     2. Feature pricing (feature_pricing table)
     3. Stripe subscription match via customer_id or stripe_customer_id
        metered  → unitPrice × usageUnits
        tiered   → tierPrice(mtdUsage) × usageUnits
        flat     → $0 per event (MRR is fixed)
     4. Default $0

  → Customer name resolution (async):
     customers.stripe_customer_id → stripe.customers.retrieve() → update name + email
