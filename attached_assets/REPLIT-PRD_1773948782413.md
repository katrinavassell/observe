# Tanso Observe — Replit Implementation Guide

## What This Is

Tanso Observe adds **feature-level economics**, a **rich pricing simulator**, and **AI-powered insights** to the existing metrics-onboarding app. Three capabilities:

1. **Feature Economics Dashboard** — see margins by feature, customer, and AI model
2. **Pricing Simulator** — model pricing changes, see per-customer churn risk and margin impact
3. **AI Insights (Premium)** — OpenAI-powered analysis of margins, pricing opportunities, and recommendations. Every AI call is tracked as an `observe_event` so we can see the cost/margin of this feature itself.

## Existing App Context

This app is already running. DO NOT rewrite the existing code. Build on top of it.

**Existing stack:** Vue 3, Vite, Vue Router, TanStack Vue Query, shadcn-vue (Radix Vue), Tailwind CSS, Chart.js, Express backend, PostgreSQL, anonymous sessions.

**Existing pages:** `/` (Pricing Analyzer), `/simulator` (basic simulator), `/data-sources` (CSV upload + Stripe sync)

**Existing backend:** Express on port 3001, proxied through Vite. All endpoints scoped by `visitorId` from session middleware.

**Existing DB tables:** `plans`, `customers`, `subscriptions`, `usage_records`, `cost_records`, `user_data_status`, `sessions`

## Phase 1: Feature Economics (Build First)

### 1.1 Create `observe_events` Table

Add this SQL migration. Run it against the PostgreSQL database:

```sql
CREATE TABLE IF NOT EXISTS observe_events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id TEXT NOT NULL,
  customer_id TEXT NOT NULL,
  feature_key TEXT NOT NULL DEFAULT 'unknown',
  event_name TEXT NOT NULL DEFAULT 'usage',
  timestamp TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  cost_amount DECIMAL(12,4) DEFAULT 0,
  cost_unit TEXT DEFAULT 'usd',
  revenue_amount DECIMAL(12,4) DEFAULT 0,
  usage_units DECIMAL(12,4) DEFAULT 0,
  model TEXT,
  model_provider TEXT,
  source TEXT NOT NULL DEFAULT 'csv',
  granularity TEXT NOT NULL DEFAULT 'monthly_aggregate',
  properties JSONB DEFAULT '{}',
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_observe_events_user_ts ON observe_events (user_id, timestamp DESC);
CREATE INDEX idx_observe_events_user_feature ON observe_events (user_id, feature_key);
CREATE INDEX idx_observe_events_user_customer ON observe_events (user_id, customer_id);
CREATE INDEX idx_observe_events_user_model ON observe_events (user_id, model) WHERE model IS NOT NULL;
CREATE INDEX idx_observe_events_user_source ON observe_events (user_id, source);
```

### 1.2 Modify Existing Upload Endpoints (Dual-Write)

When data is uploaded via CSV or synced from Stripe, ALSO write to `observe_events`. Add this logic to the existing endpoints in `server/index.ts`:

**For `/data/upload/costs`:** After inserting into `cost_records`, also insert into `observe_events`:
- `feature_key` = `record.provider` or `'infrastructure'`
- `event_name` = `'cost'`
- `cost_amount` = `record.cost`
- `source` = `'csv'`
- `granularity` = `'monthly_aggregate'`
- `customer_id` = `record.customer_id` or `'_aggregate'`

**For `/data/upload/usage`:** After inserting into `usage_records`, also insert into `observe_events`:
- `feature_key` = `record.metric` or `record.metric_key`
- `event_name` = `'usage'`
- `usage_units` = `record.value` or `record.metric_value`
- `source` = `'csv'`
- `granularity` = `'monthly_aggregate'`

**For `/data/upload/revenue`:** After inserting plans/customers/subscriptions, also insert into `observe_events`:
- `feature_key` = `'subscription'`
- `event_name` = `'revenue'`
- `revenue_amount` = subscription MRR
- `source` = `'csv'`

**For `/stripe/sync`:** After syncing, also insert `observe_events` with `source: 'stripe'`.

**For `/data/sample`:** Also generate sample `observe_events` with realistic feature/model data. Use these features:
- `api_requests` (high volume, low cost)
- `pdf_generation` (medium volume, high cost)
- `ai_summarization` (medium volume, uses gpt-4o)
- `image_generation` (low volume, high cost, uses dall-e-3)
- `search` (high volume, low cost)

And these models: `gpt-4o`, `gpt-4o-mini`, `claude-sonnet-4`, `dall-e-3`

**For `/data/clear`:** Also delete from `observe_events`.

### 1.3 Add New API Endpoints

Add these to `server/index.ts`:

```
GET /events          - List events with pagination + filters (customer_id, feature_key, model, date range)
GET /events/by-feature  - Aggregate: SUM(cost_amount), SUM(revenue_amount), SUM(usage_units) GROUP BY feature_key
GET /events/by-customer - Aggregate: SUM(cost_amount), SUM(revenue_amount) GROUP BY customer_id
GET /events/by-model    - Aggregate: SUM(cost_amount), COUNT(*), SUM(usage_units) GROUP BY model, model_provider
GET /features/:key      - Events + aggregations for a single feature (timeseries by month)
GET /customers/:id      - Events + aggregations for a single customer
```

All endpoints must be scoped by `user_id` (from `ensureVisitor` middleware).

## Phase 3: AI Insights (Premium Feature)

### What It Does

AI-powered analysis that reads the user's data and generates actionable insights about margins, pricing, and customer health. Powered by OpenAI (`gpt-4o-mini` for cost efficiency).

**This feature dogfoods Observe itself:** every OpenAI API call is logged as an `observe_event` with `feature_key: 'ai_insights'`, so we can track the cost/margin of this premium feature through the same dashboard.

### 3.1 Environment

Add `OPENAI_API_KEY` as a Replit secret. Install the `openai` npm package.

### 3.2 Backend: AI Insights Endpoint

Add to `server/index.ts`:

```
POST /insights/generate  - Generate AI insights from user's data
GET  /insights            - Get previously generated insights
```

**`POST /insights/generate` flow:**
1. Fetch the user's aggregated data (by-feature margins, top customers, model costs)
2. Build a prompt with the data summary
3. Call OpenAI `gpt-4o-mini` with the prompt
4. **Log the OpenAI call as an observe_event:**
   ```typescript
   await pool.query(
     `INSERT INTO observe_events (user_id, customer_id, feature_key, event_name, cost_amount, cost_unit, model, model_provider, source, granularity)
      VALUES ($1, '_system', 'ai_insights', 'inference', $2, 'usd', 'gpt-4o-mini', 'openai', 'sdk', 'event')`,
     [visitorId, openaiCostEstimate]
   )
   ```
5. Save insights to a new table and return them

**Cost estimation:** Track token usage from the OpenAI response. Approximate cost = `(prompt_tokens * 0.15 + completion_tokens * 0.60) / 1_000_000` for gpt-4o-mini.

### 3.3 Database Table

```sql
CREATE TABLE IF NOT EXISTS ai_insights (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id TEXT NOT NULL,
  insight_type TEXT NOT NULL,  -- 'margin_alert', 'pricing_opportunity', 'cost_optimization', 'customer_risk'
  title TEXT NOT NULL,
  description TEXT NOT NULL,
  severity TEXT DEFAULT 'info',  -- 'critical', 'warning', 'info', 'positive'
  feature_key TEXT,              -- Related feature (if applicable)
  customer_id TEXT,              -- Related customer (if applicable)
  metadata JSONB DEFAULT '{}',
  tokens_used INTEGER,
  cost_usd DECIMAL(10,6),
  created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_insights_user ON ai_insights (user_id, created_at DESC);
```

### 3.4 Prompt Design

Send OpenAI a structured summary of the user's data:

```
You are a SaaS pricing analyst. Analyze this data and return 3-5 actionable insights.

Feature margins:
- api_requests: revenue $4,200, cost $890, margin 79% (healthy)
- ai_summarization: revenue $1,100, cost $2,340, margin -113% (NEGATIVE)
- pdf_generation: revenue $800, cost $650, margin 19% (low)

Customer summary:
- 45 customers, avg margin 34%
- 3 customers are underwater (negative margin)

Model costs:
- gpt-4o: $1,800/month, 12,000 calls
- gpt-4o-mini: $340/month, 45,000 calls

Return JSON array of insights with: type, title, description, severity, feature_key (if relevant).
```

### 3.5 Frontend: Insights Panel

Add an "AI Insights" section to the home page (`/`) or as a dedicated `/insights` page:
- "Generate Insights" button (calls `POST /insights/generate`)
- Loading state while OpenAI processes
- Card list of insights with severity badges (critical=red, warning=amber, info=blue, positive=green)
- Each insight card shows: title, description, related feature/customer link
- Footer: "Powered by AI · Cost: $0.003 this analysis"

### 3.6 The Dogfooding Payoff

Once AI Insights is built, the user (us) can go to `/features` and see:
- `ai_insights` feature with its total cost (sum of all OpenAI calls)
- If we charge a premium for it, we can see revenue vs cost = **the margin of the AI feature itself**
- Then use the Simulator to model: "What if we charge $10/month for AI Insights? Is it profitable?"
